Operational Efficiency & Throughput

The system manages three distinct payment workflows: Automated RFID (E-Tag), Dynamic QR Fallback, and Administrative Overrides. By prioritizing E-Tag throughput, we reduce plaza congestion by up to 85% compared to legacy cash-only lanes.

Silent Security Interception Flow

When the ANPR cameras detect a plate flagged in the excise.vehicles schema as 'Stolen', the system executes a silent workflow. This prevents suspect alertness while ensuring law enforcement is dispatched with real-time vector data.

Step 01: Capture

ANPR Camera Scan

API receives license plate and E-Tag signature in a unified payload.

Step 02: Verification

PostgreSQL Schema Cross-Ref

Dual-Factor validation checks if Tag ID matches Plate Number within 10ms.

Step 03: Flag (Silent)

Trigger Execution

Database trigger changes passage state to 'None_Silent_Flag' for stolen assets.

Step 04: Action

Barrier Activation

Barrier opens to prevent traffic jams; Law Enforcement dashboard updates instantly.

Security Alert Categorization

The majority of alerts involve E-Tag mismatches (potential fraud), followed by the high-priority stolen vehicle interceptions.

Backend Resilience & Architecture

Technical Prowess

By utilizing a monolithic database architecture logically partitioned into three schemas (Excise, Toll, Payment), we eliminate the network overhead typical of microservices while maintaining strict data boundaries.

FastAPI Performance

Asynchronous task handling ensures consistent performance during peak traffic (10,000+ requests/sec).

SQLAlchemy 2.0 Logic

Native ORM integration prevents SQL injection and ensures financial precision via Decimal types.

PostgreSQL Constraints

Foreign keys and triggers enforce business rules at the data layer, acting as a secondary security barrier.
